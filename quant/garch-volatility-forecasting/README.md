# GARCH(1,1) Volatility Forecasting

Fits a GARCH(1,1) model to a synthetic return series that genuinely exhibits volatility
clustering (generated from a known GARCH process), recovers the underlying parameters,
compares GARCH's fitted volatility against constant and rolling-window estimates, and
produces a forward volatility forecast.

For the theory behind GARCH and how it complements the VaR/CVaR project's distributional
findings, see [`theory/garch_theory.md`](theory/garch_theory.md).

## What the code does

1. **`generate_data.py`**: generates 1,500 days of synthetic returns from a known GARCH(1,1)
   process (alpha=0.09, beta=0.88), saving both the returns and the "true" underlying
   volatility path (known here since we control the data generation) to `data/`.
2. **`garch_model.py`**:
   - **Fits a GARCH(1,1) model** via the `arch` library's maximum-likelihood estimation.
   - Compares the fitted parameters against the known true values as a correctness check.
   - Computes **constant**, **rolling-window (20-day)**, and **GARCH conditional** volatility
     estimates side by side.
   - **Evaluates each method's accuracy** against next-day realized |return| (correlation and MAE).
   - Produces a **20-day forward volatility forecast**, showing the decay from current
     conditional volatility toward the long-run unconditional level.
   - Outputs a volatility comparison chart, a forecast chart, and detailed CSVs.

## Data note

Live market data wasn't reachable in this environment (network/SSL restriction), so this
project uses **reproducible synthetic data generated from a known GARCH process** — letting
the fitted model's recovered parameters be checked directly against ground truth, which
wouldn't be possible with real data (where the true generating process is never actually known).

## How to run

```bash
cd code
pip install -r requirements.txt
python generate_data.py
python garch_model.py
```

Outputs are written to `output/`.

## Key results (from the included synthetic data)

- **Fitted GARCH parameters (alpha=0.095, beta=0.833) closely track the true generating
  values (0.09, 0.88)** — a genuine correctness verification for the fitting procedure.
- **Persistence (alpha+beta) of ~0.93**, close to the true 0.97 — high persistence, typical
  for real daily equity/asset returns, meaning volatility shocks decay slowly.
- **Both GARCH (correlation 0.226) and rolling volatility (0.202) meaningfully outperform
  constant volatility (~0.000)** at predicting next-day realized |return| — confirming
  time-varying volatility estimation is essential, with GARCH showing an edge over the
  simpler rolling-window approach.
