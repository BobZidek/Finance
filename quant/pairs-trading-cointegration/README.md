# Pairs Trading / Statistical Arbitrage with Cointegration Testing

Tests whether two synthetic assets (constructed to genuinely be cointegrated) actually show
statistically significant cointegration via the Engle-Granger two-step method, cross-checked
against statsmodels' properly-adjusted `coint()` test — then backtests a z-score mean-reversion
trading rule on the spread.

For the theory behind cointegration and **an important subtlety this project's own results
reveal about testing methodology**, see
[`theory/cointegration_and_pairs_trading.md`](theory/cointegration_and_pairs_trading.md).

## What the code does

1. **`generate_data.py`**: generates two synthetic price series that share a common
   non-stationary trend, with their spread following a stationary (mean-reverting) AR(1)
   process — genuinely cointegrated by construction — saved to `data/pair_prices.csv`.
2. **`pairs_trading_model.py`**:
   - Runs the **Engle-Granger two-step test**: OLS regression for the hedge ratio, then an
     Augmented Dickey-Fuller test on the residual spread.
   - **Cross-checks against `statsmodels.tsa.stattools.coint()`**, which uses the correct
     adjusted critical values for this exact test — and reports both results even when they disagree.
   - Computes the spread's rolling z-score and generates trading positions (enter at ±2
     std devs, exit at ±0.5).
   - Backtests the strategy (shifted one day to avoid look-ahead bias), computing Sharpe,
     max drawdown, and trade count.
   - Outputs a 3-panel chart (prices, spread, z-score with entry/exit bands) and an equity curve.

## Data note

Live market data wasn't reachable in this environment (network/SSL restriction), so this
project uses a **reproducible synthetic pair** (fixed random seed), deliberately constructed
to be cointegrated so the test methodology can be verified against a known-true relationship.

## How to run

```bash
cd code
pip install -r requirements.txt
python generate_data.py
python pairs_trading_model.py
```

Outputs are written to `output/`.

## Key results (from the included synthetic data)

- **Hedge ratio (OLS beta): 0.6578.**
- **The naive ADF-on-residuals test says "cointegrated" (p = 0.0485)**, but the **properly
  adjusted `coint()` test says "not clearly cointegrated" (p = 0.142)** — on data constructed
  to genuinely be cointegrated. The theory doc explains why this happened (a known
  methodological trap with naive critical values, compounded by a near-unit-root spread that's
  statistically hard to distinguish from non-stationary in a finite sample) and why the
  correct test's conclusion should be trusted over the naive one.
- The backtest still produced a **1.04 Sharpe ratio and 33.75% total return over 7 trades** —
  a genuinely encouraging result on this specific path, but one that, given the ambiguous
  formal test, a real desk should treat as a candidate for further validation rather than a
  confirmed statistical edge.
