# Hidden Markov Model (HMM) Regime Detection

Fits a 2-state Gaussian HMM to a synthetic return series generated from a known, genuinely
persistent 2-regime Markov process ("Calm" and "Turbulent"), recovers the hidden state
parameters and transition matrix, decodes the most likely regime path, checks decoding
accuracy against the known true labels, and demonstrates a regime-aware exposure strategy —
**with an explicit, honest caveat about its in-sample nature**.

For the theory behind HMMs and **why the backtest result should be read carefully**, see
[`theory/hmm_theory.md`](theory/hmm_theory.md).

## What the code does

1. **`generate_data.py`**: generates 1,500 days of synthetic returns from a known 2-state
   Markov process with persistent transitions (Calm: low vol/positive drift; Turbulent: high
   vol/negative drift), saving both returns and the true regime labels.
2. **`hmm_model.py`**:
   - Fits a 2-state Gaussian HMM via `hmmlearn`'s Baum-Welch algorithm.
   - Aligns the HMM's arbitrary state indices to the "Calm"/"Turbulent" labels by comparing
     fitted means.
   - Decodes the most likely regime path via the Viterbi algorithm.
   - Checks decoding accuracy against the known true regime labels.
   - Runs a regime-aware exposure strategy (50% exposure when decoded as Turbulent, 100%
     otherwise) and compares it against buy-and-hold — **explicitly flagged as an in-sample
     demonstration, not a valid backtest**.
   - Outputs a regime-detection chart, a backtest equity curve chart, and detailed CSVs.

## Data note

Live market data wasn't reachable in this environment (network/SSL restriction), so this
project uses a **reproducible synthetic dataset generated from a known regime-switching
process** — letting the fitted HMM's recovered parameters be checked directly against ground
truth.

## How to run

```bash
cd code
pip install -r requirements.txt
python generate_data.py
python hmm_model.py
```

Outputs are written to `output/`.

## Key results (from the included synthetic data)

- **Regime decoding accuracy: 97.2%** against the known true regime labels — the HMM
  genuinely recovers the underlying 2-regime structure from return data alone.
- **Fitted volatilities closely match true values** (Calm: 0.834% vs. true 0.800%; Turbulent:
  2.591% vs. true 2.500%); fitted **means** show more estimation noise, especially for the
  smaller-sample Turbulent regime — an expected, honestly-reported statistical limitation.
- **The regime-aware strategy shows a materially better risk profile** in this demonstration
  (Sharpe 1.04 vs. 0.60, max drawdown -19.0% vs. -37.5% for buy-and-hold) — but **this is an
  in-sample result** (the HMM was fit on the full dataset at once), explicitly flagged in the
  theory doc as not a valid causal backtest.
