# ML Return-Prediction Model with Realistic Transaction Costs

Engineers momentum, volatility, and mean-reversion features from a synthetic single-asset
price series (with a known, injected — deliberately generous — predictive relationship), trains
Ridge and Random Forest models on a strict time-ordered train/test split, and backtests a
sign-based trading strategy net of realistic transaction costs.

For the theory behind the pipeline, a real bug this project's own development caught, and
**an honest, deliberately unflattering finding about why single-asset ML return prediction is
hard**, see [`theory/ml_prediction_theory.md`](theory/ml_prediction_theory.md).

## What the code does

1. **`generate_data.py`**: generates 5 years of synthetic daily prices where next-day return
   is a noisy function of the prior day's momentum and mean-reversion signals — a genuine,
   known relationship for the models to attempt to recover.
2. **`ml_model.py`**:
   - Engineers 4 features (5-day and 20-day momentum, 20-day realized volatility, mean-reversion
     z-score) using only same-day-or-earlier information.
   - **Splits train/test chronologically** (no shuffling — critical to avoid look-ahead bias).
   - Trains a **Ridge regression** (linear) and a **Random Forest** (nonlinear) on the training set.
   - Evaluates out-of-sample directional accuracy and Information Coefficient.
   - Backtests a sign-based long/short strategy from each model's predictions, with a 5bps
     per-trade transaction cost, comparing gross vs. net vs. buy-and-hold.
   - Outputs an evaluation table, a backtest performance table, equity curve charts, and a
     gross-vs-net comparison chart showing the transaction cost drag.

## Data note

Live market data wasn't reachable in this environment (network/SSL restriction), so this
project uses a **reproducible synthetic price series** (fixed random seed) with a known,
generously-sized injected signal — see the theory doc for why even this generous signal
proved hard to reliably exploit.

## How to run

```bash
cd code
pip install -r requirements.txt
python generate_data.py
python ml_model.py
```

Outputs are written to `output/`.

## Key results (from the included synthetic data)

- **Directional accuracy hovered at 50-51%** for both models (essentially a coin flip), and
  **Information Coefficient was weak** (Ridge 0.033, Random Forest 0.003) — despite the data
  containing a known, deliberately generous injected signal.
- **Both models' sign-based strategies lost money**, even before transaction costs (Ridge:
  -19.3%, Random Forest: -15.8% gross total return over the ~19-month test period), while
  **buy-and-hold gained +76.1%** over the same period.
- **This is presented as a genuine, useful finding**, not hidden: it demonstrates that a small
  positive IC doesn't reliably translate into a profitable simple trading rule, and that
  single-asset time-series return prediction is structurally much harder than the
  cross-sectional ranking approach in
  [`quant/multi-factor-stock-ranking`](../multi-factor-stock-ranking) — see the theory doc for
  the full explanation of why.
- A real unit-mismatch bug in the mean-reversion feature was caught and fixed during
  development — documented in the theory doc rather than silently corrected.
